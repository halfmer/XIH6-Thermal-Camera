# 热成像数据传输协议 & 平台指南

> 上位机统一解析格式，ESP32 和 STM32 发送端协议完全一致。

---

## 一、二进制帧协议（通用）

### 帧结构

```
┌────────┬────────┬──────┬──────────┬───────┬────────┬───────────┬──────────┬──────────┐
│ 0xAA   │ 0x55   │ type │ frame_id │ width │ height │ pixel_len │  pixels  │ checksum │
│ 1 B    │ 1 B    │ 1 B  │   2 B    │  2 B  │  2 B   │    4 B    │   N B    │   2 B    │
└────────┴────────┴──────┴──────────┴───────┴────────┴───────────┴──────────┴──────────┘
```

| 偏移 | 字节数 | 字段 | 类型 | 说明 |
|------|--------|------|------|------|
| 0 | 1 | sync0 | u8 | 固定 `0xAA` |
| 1 | 1 | sync1 | u8 | 固定 `0x55` |
| 2 | 1 | type | u8 | `0x01` = raw 16-bit thermal |
| 3 | 2 | frame_id | u16 BE | 帧序号，自增，上位机用于统计丢失 |
| 5 | 2 | width | u16 BE | 图像宽度，LEPTON 3.5 = `160` |
| 7 | 2 | height | u16 BE | 图像高度，LEPTON 3.5 = `120` |
| 9 | 4 | pixel_len | u32 BE | 像素数据字节数 = width × height × 2 |
| 13 | N | pixels | u16 BE[] | 原始像素数组，每像素 16-bit 大端 |
| 13+N | 2 | checksum | u16 BE | 从 sync0 到 pixels 末字节的累加和 |

### 帧大小

| 传感器 | 分辨率 | 像素字节 | 帧总字节 |
|--------|--------|----------|----------|
| LEPTON 3.5 | 160×120 | 38,400 B | **38,415 B** |
| LEPTON 3.1R | 160×120 | 38,400 B | **38,415 B** |
| 自定义 | W×H | W×H×2 | W×H×2 + 15 |

### 校验和算法

```c
// 简单累加，溢出截断到 16-bit
uint16_t checksum = 0;
for (int i = 0; i < total_bytes_before_checksum; i++)
    checksum += buffer[i];
// checksum 以 uint16 BE 写入帧尾
```

## 二、ESP32 发送端

### 硬件连接

```
ESP32-S3-N16R8          USB-UART (内建)
┌──────────────┐        ┌──────────┐
│  TX  (IO43)  ├───────►│  RX      │
│  RX  (IO44)  │◄───────│  TX      │
│  EN          │◄─DTR───│          │  ← 自动下载电路
│  IO0         │◄─RTS───│          │  ← 自动下载电路
│  GND         ├────────│  GND     │
└──────────────┘        └────┬─────┘
                             │
                         上位机 USB
```

### 关键参数

| 项目 | ESP32-S3 |
|------|----------|
| **波特率** | `921600`（推荐），最高 `3000000` |
| **数据位** | 8 |
| **停止位** | 1 |
| **校验位** | 无 |
| **流控** | 无 |
| **DTR** | 连接 EN 引脚，上位机勾选可复位芯片 |
| **RTS** | 连接 IO0 引脚，上位机勾选可进入下载模式 |
| **UART 编号** | 可使用任意 UART，推荐 UART0 (USB-JTAG 自带) 或 UART1 |

### 发送代码（Arduino / ESP-IDF）

```cpp
#define THERMAL_W  160
#define THERMAL_H  120

static uint16_t g_frame_id = 0;

void sendThermalFrame(uint16_t *pixel_buf) {
    uint32_t pixel_len = THERMAL_W * THERMAL_H * 2;
    uint16_t cksum = 0;
    uint8_t b;

    // Sync
    Serial.write(0xAA); cksum += 0xAA;
    Serial.write(0x55); cksum += 0x55;

    // Type
    Serial.write(0x01); cksum += 0x01;

    // frame_id BE
    b = g_frame_id >> 8;  Serial.write(b); cksum += b;
    b = g_frame_id & 0xFF; Serial.write(b); cksum += b;
    g_frame_id++;

    // width BE
    b = THERMAL_W >> 8;  Serial.write(b); cksum += b;
    b = THERMAL_W & 0xFF; Serial.write(b); cksum += b;

    // height BE
    b = THERMAL_H >> 8;  Serial.write(b); cksum += b;
    b = THERMAL_H & 0xFF; Serial.write(b); cksum += b;

    // pixel_len BE
    b = pixel_len >> 24; Serial.write(b); cksum += b;
    b = pixel_len >> 16; Serial.write(b); cksum += b;
    b = pixel_len >> 8;  Serial.write(b); cksum += b;
    b = pixel_len & 0xFF; Serial.write(b); cksum += b;

    // Pixel data
    uint8_t *p = (uint8_t *)pixel_buf;
    for (uint32_t i = 0; i < pixel_len; i++) {
        Serial.write(p[i]);
        cksum += p[i];
    }

    // Checksum BE
    b = cksum >> 8;  Serial.write(b);
    b = cksum & 0xFF; Serial.write(b);
}
```

### ESP32 注意事项

| ⚠️ 注意 | 说明 |
|----------|------|
| **别混日志** | 热成像帧**必须独占一个串口**，不要和 `Serial.print()` 共用。日志混入会破坏帧同步，上位机报 `checksum mismatch` |
| **VoSPI 数据即 BE** | LEPTON 通过 VoSPI 输出的像素已经是 uint16 大端，**直接搬进帧里**，不要做字节序转换 |
| **DTR/RTS** | 上位机 DTR/RTS 复选框映射到 ESP32 的 EN/IO0，可用于手动复位和下载。正常传输时**不要勾选** |
| **堆栈空间** | 一帧 38KB+，ESP32 默认栈不够，帧 buffer 用 `heap_caps_malloc` 或 `static` 全局分配 |
| **帧率上限** | 921600 bps ÷ (38415×10) ≈ **2.4 fps** 理论极限。要更高帧率用 2M/3M 波特率 |

## 三、STM32H743 + CH340 发送端

### 硬件连接

```
STM32H743XIH6           CH340C
┌──────────────┐        ┌──────────┐
│ UART4 PA1*   ├───────►│  RXD     │
│ UART4 PA0*   │◄───────│  TXD     │
│  GND         ├────────│  GND     │
│  3.3V        │        │  3.3V    │
└──────┬───────┘        └────┬─────┘
       │                     │
  ┌────┴────┐            上位机 USB
  │ LEPTON  │
  │ 3.5     │  ← VoSPI
  └─────────┘
```

*本板 PA0/PA1 与 CH340C 方向在原理图上对调，STM32 固件运行时开启 UART4 `CR2.SWAP`，
所以外部物理连接仍按原理图，不需要飞线对调。USART1(PA9/PA10) 只做调试日志。

### 关键参数

| 项目 | STM32H743 + CH340 |
|------|-------------------|
| **波特率** | `2000000`（当前 STM32 固件固定），CH340C 按 2 Mbps 使用 |
| **数据位** | 8 |
| **停止位** | 1 |
| **校验位** | 无 |
| **流控** | 无 |
| **DTR/RTS** | CH340 支持，但通常不接，上位机无需勾选 |
| **UART** | 当前板级链路为 UART4 (PA0/PA1 + CR2.SWAP)；USART1(PA9/PA10) 为调试日志 |
| **CH340 驱动** | Windows 需装驱动，设备管理器显示 `USB-SERIAL CH340` |

### 发送代码（STM32 HAL）

```c
#define THERMAL_W  160
#define THERMAL_H  120

static uint16_t g_frame_id = 0;
static UART_HandleTypeDef *g_huart = &huart4;  // CH340C/UART4 热成像串口

void sendThermalFrame(uint8_t *frame_buffer) {
    uint32_t pixel_len = THERMAL_W * THERMAL_H * 2;
    uint16_t cksum = 0;
    uint8_t buf[16];

    // Sync
    buf[0] = 0xAA; buf[1] = 0x55;
    HAL_UART_Transmit(g_huart, buf, 2, HAL_MAX_DELAY);
    cksum += 0xAA + 0x55;

    // Header [type(1) + frame_id(2) + w(2) + h(2) + pixel_len(4)]
    uint16_t fid = g_frame_id++;
    uint32_t pl  = pixel_len;
    buf[0]  = 0x01;
    buf[1]  = (fid >> 8) & 0xFF;
    buf[2]  = fid & 0xFF;
    buf[3]  = (THERMAL_W >> 8) & 0xFF;
    buf[4]  = THERMAL_W & 0xFF;
    buf[5]  = (THERMAL_H >> 8) & 0xFF;
    buf[6]  = THERMAL_H & 0xFF;
    buf[7]  = (pl >> 24) & 0xFF;
    buf[8]  = (pl >> 16) & 0xFF;
    buf[9]  = (pl >> 8) & 0xFF;
    buf[10] = pl & 0xFF;
    HAL_UART_Transmit(g_huart, buf, 11, HAL_MAX_DELAY);
    for (int i = 0; i < 11; i++) cksum += buf[i];

    // Pixels
    HAL_UART_Transmit(g_huart, frame_buffer, pixel_len, HAL_MAX_DELAY);
    for (uint32_t i = 0; i < pixel_len; i++) cksum += frame_buffer[i];

    // Checksum
    buf[0] = (cksum >> 8) & 0xFF;
    buf[1] = cksum & 0xFF;
    HAL_UART_Transmit(g_huart, buf, 2, HAL_MAX_DELAY);
}
```

### STM32 注意事项

| ⚠️ 注意 | 说明 |
|----------|------|
| **HAL_MAX_DELAY 会阻塞** | 2 Mbps 下 38KB 帧耗时约 192ms，阻塞发送会让采集与发送串行。后续可用 DMA + 乒乓缓冲提升帧率 |
| **时钟要配准** | H743 UART 波特率依赖 PCLK 分频。当前 UART4 内核时钟 120MHz，2 Mbps = 120M/60，误差 0% |
| **像素字节序** | LEPTON VoSPI 输出 uint16 BE，直接放入帧。如果中间做了 `ntohs` 或手动转了 LE，**记得转回 BE** |
| **CH340 驱动** | Windows 设备管理器应显示 `USB-SERIAL CH340`；若完全不枚举，先查 CH340C 供电和 Type-C CC/VBUS |
| **别混 printf** | 和 ESP32 一样——热成像串口**只发帧**，日志走另一个 UART |
| **帧 buffer 大** | 38KB 栈上分配会爆，用全局 `static uint8_t` 或 `__attribute__((section(".ram_dtcm")))` 放 DTCM |

## 四、ESP32 vs STM32 对比

| 项目 | ESP32-S3 | STM32H743 + CH340 |
|------|----------|-------------------|
| **USB 转串口** | 内建 (USB-JTAG) | 外置 CH340 |
| **上位机识别** | `COMx — USB JTAG/serial debug` | `COMx — USB-SERIAL CH340 [CH340 — STM32/ESP32]` |
| **自动下载** | ✅ DTR→EN, RTS→IO0 | ❌ 需手动 BOOT0/RESET |
| **波特率上限** | 3 Mbps | 3 Mbps (H743 硬件) / 2 Mbps (CH340 稳定) |
| **当前 STM32 帧率** | 取决于发送端 | 2 Mbps 理论约 5.2 fps，当前阻塞实现约 3 fps |
| **DMA 发送** | ESP-IDF `uart_write_bytes` 自带缓冲 | 需手动配置 `HAL_UART_Transmit_DMA` |
| **帧数据源** | ESP32 直接接 Lepton VoSPI | STM32 接 Lepton VoSPI，再通过 UART 转 CH340 |
| **推荐结构** | VSPI → ESP32 → USB → 上位机 | VSPI → STM32 → UART → CH340 → 上位机 |

## 五、上位机操作步骤

1. 插上设备 → 打开 `UART 上位机`
2. 点 **刷新** → 选对应 COM 口
   - ESP32: `COMx — USB JTAG/serial debug`
   - STM32: `COMx — USB-SERIAL CH340`（绿色高亮）
3. STM32/CH340 链路波特率选 **2000000**（ESP32 按其固件配置选择）
4. 工作模式切到 **热成像**
5. 点 **打开串口** → 接收帧，画面自动显示
6. 色表可选：铁红 / 白热 / 黑热 / 彩虹 / 熔岩 / 极地
